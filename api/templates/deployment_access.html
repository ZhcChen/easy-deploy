面向业务项目 CI、AI 调用方和运维人员的部署配置、版本包、OpenAPI 与发布执行规范。

接入边界 OpenAPI 只登记版本

外部项目上传部署单元包并创建应用版本，不直接创建应用、不改配置、不触发部署。

发布入口 后台手动启动

运维在应用列表或详情中选择版本、环境和 normal/force 模式后启动部署。

多模块项目 一个应用，多部署单元

例如 qfy-voucher-hub 作为一个应用，gateway、api、admin、worker 等作为部署单元。

接入步骤

完整字段、示例 curl、脚本契约和清理策略以独立文档为准。

1. 后台建应用与部署单元

配置 Compose、独立 .env、目标节点、脚本、健康检查和环境配置版本。

2. CI 上传单元发布包

调用 POST /api/v1/apps/{app_key}/units/{unit_key}/releases，上传 version 与包文件。

3. CI 创建应用版本

调用 POST /api/v1/apps/{app_key}/releases，把多个单元版本拼成不可变 manifest。

4. 运维手动部署

选择 normal 或 force。normal 只执行目标状态不一致或不健康的单元；force 重新执行所有启用单元。
